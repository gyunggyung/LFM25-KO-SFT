# round15 RAG Context v5 구축ㆍ검증 방식 설명

작성일: 2026-06-29

이 문서는 `round15_rag_contexts_v5_20260629`를 어떻게 만들고 검증하는지 추적하기 위한 작업 문서다. 실제 문제 풀이용 모델에는 이 문서를 제공하지 않는다.

## 현재 진행 상태

- 대상: 제15회 변호사시험 150문항
- v5 디렉터리: `round15_rag_contexts_v5_20260629`
- Antigravity 지시문: `round15_antigravity_context_v5_20260629.md`
- 문제 원문 교체: 150/150 완료
- 선지 분해 교체: 150/150 완료
- q-context 정답 누출 스캔: 현재 0건
- 명시적 `v5 수동 검증 보강 포인트` 섹션: 현재 150/150개
- 남은 명시적 수동 보강 대상: 현재 0개

## v5의 목표

v5는 이름만 v5이고 운영상으로는 v4.1에 가깝다. v4의 구조와 잘 맞은 자동 후보는 최대한 살리되, 다음 문제를 고치는 데 초점을 둔다.

1. 문제 본문과 선지가 깨진 상태로 들어간 파일을 `15th_split` 원문 기준으로 정리한다.
2. 과거 오답 문항의 원인을 명시적으로 반영한다.
3. 자동 검색 후보가 엉뚱한 사건ㆍ법령을 끌고 온 경우, 수동 검증 섹션에서 우선순위를 뒤집는다.
4. 정답 번호나 공식 해설을 넣지 않는다.
5. 그 파일만 읽어도 법리 추론으로 다지선다를 풀 수 있을 정도의 근거를 제공한다.

## v1-v4 대비 차이

v1-v4의 핵심 한계는 다음과 같았다.

- 자동 리트리버 후보가 실제 선지 법리와 빗나간 경우가 있었다.
- 문제 원문 개행과 선지 분해가 깨끗하지 않은 파일이 있었다.
- 맞은 근거와 틀린 근거가 같은 파일 안에서 섞이면 모델이 자동 후보를 과신할 수 있었다.
- 과거 오답 원인이 파일 안에 명시되지 않아, 같은 함정에서 다시 틀릴 위험이 있었다.

v5 방식은 다음을 추가한다.

- `15th_split` JSON을 원문 기준으로 삼아 `## 문제`와 `## 선지 분해`를 150개 전부 재생성한다.
- 각 고위험 문항에 `## v5 수동 검증 보강 포인트`를 넣어, 자동 후보보다 우선 적용할 법리를 따로 명시한다.
- 법령ㆍ판례 근거는 공식 법령정보센터, 헌법재판소, 대법원/법원도서관 계열 자료와 로컬 `legalize-kr`, `precedent-kr`를 대조한다.
- q-context 내부에 `정답`, `answer`, `gold`, `모범답안`, `채점` 같은 누출성 표현이 들어가지 않도록 반복 스캔한다.
- 실제 풀이 지시문은 `15th_split`과 v5 q-context만 보게 하고, 정답ㆍ해설ㆍ검증ㆍ로컬 판례 DBㆍ웹 검색은 금지한다.

## 원문 교체 방식

원문 교체는 다음 스크립트로 수행한다.

- 스크립트: `round15_v5_audit/replace_problem_blocks_from_split.py`
- 입력:
  - `15th_split/public_law`
  - `15th_split/criminal_law`
  - `15th_split/civil_law`
- 출력:
  - `round15_rag_contexts_v5_20260629/q*_공법.md`
  - `round15_rag_contexts_v5_20260629/q*_형사법.md`
  - `round15_rag_contexts_v5_20260629/q*_민사법.md`

이 단계는 문제 문장, 보기, ㄱㄴㄷ식 선지, ①-⑤ 선택지를 JSON 원문에서 평문으로 재구성한다. 정답 필드는 사용하지 않는다.

## 수동 검증 방식

각 문항은 다음 순서로 본다.

1. `15th_split` 원문과 q-context 문제/선지 분해가 같은지 확인한다.
2. v4 자동 후보 중 실제 쟁점과 맞는 법령ㆍ판례는 유지한다.
3. 자동 후보가 빗나가거나 과거 오답과 연결되는 경우 수동 보강 섹션을 추가한다.
4. 공식 법령 또는 공식 판례 링크를 붙인다.
5. 선지별로 필요한 법리만 압축해서 적는다.
6. 최종 답 번호는 적지 않는다.
7. 누출 스캔을 돌린다.

검증에 쓰는 대표 명령은 다음과 같다.

```bash
rg -l '^## v5 수동 검증 보강 포인트' round15_rag_contexts_v5_20260629/q*_*.md | wc -l
rg -n "정답|answer|gold|모범답안|해설상|채점|score" round15_rag_contexts_v5_20260629/q*_*.md
```

## 근거 수집 출처

v5 구축 중에는 다음 자료를 사용한다.

- 로컬 법령: `/home/work/.projects/LLM-OS-Models/Terminal/HRM-Text/legalize-kr/kr/...`
- 로컬 판례: `precedent-kr/...`
- 공식 법령정보센터: `https://www.law.go.kr`
- 헌법재판소 결정/주요판례: `https://www.ccourt.go.kr`
- 필요 시 Google 검색 또는 웹 검색

단, 이 자료들은 v5 근거를 만들 때만 사용한다. 실제 풀이 지시문에서는 모델이 로컬 법령ㆍ판례 DB, 웹, 정답 파일, 기존 풀이 파일을 열지 못하도록 금지한다.

## 정답 누출 방지 원칙

q-context에는 다음을 넣지 않는다.

- 공식 정답 번호
- `정답은 n번` 식의 문장
- 공식 해설 원문
- 채점 결과
- 과거 모델의 선택 번호

허용하는 것은 선지별 법리 판단에 필요한 근거다. 다지선다 문제는 결국 각 선지의 법리 참ㆍ거짓을 추론한 뒤 선택지 조합을 고르는 구조이므로, q-context에는 그 추론 재료를 넣는다.

## 왜 현재 방식이 최선인지

이 방식이 현재 조건에서 가장 안정적이다.

- 문제 원문은 `15th_split`로 고정해서 입력 노이즈를 없앤다.
- 법리 근거는 q-context 안에 넣되, 정답 번호는 넣지 않는다.
- 자동 리트리버의 장점인 폭넓은 후보 수집은 살리고, 틀릴 수 있는 부분은 수동 섹션으로 제어한다.
- 실제 풀이 단계에서는 외부 자료 접근을 막아, 리트리버 근거 자체의 품질을 테스트할 수 있다.
- 전수 수동 검증을 하면 “잘 맞은 파일은 최소 수정, 틀릴 가능성이 있는 파일은 강하게 보강”하는 방식으로 비용과 정확도를 균형 있게 맞출 수 있다.
