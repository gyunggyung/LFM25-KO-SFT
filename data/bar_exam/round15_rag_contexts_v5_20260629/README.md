# 제15회 변호사시험 RAG 컨텍스트 v5

작성일: 2026-06-29

이 폴더는 `round15_rag_contexts_v4_20260625`를 그대로 복사한 뒤, 이름은 `v5`로 두되 실제 작업 범위는 `v4.1`처럼 운영하는 안정화판입니다. 목표는 이미 맞던 문항을 불필요하게 건드리지 않고, 리트리버 산출물의 충돌ㆍ누락ㆍ선지 매칭 위험을 전 문항 기준으로 검증하는 것입니다.

문제와 선택지는 포함하지만, 문항별 공식 선택 번호나 평가표는 넣지 않습니다. 모델이 번호를 외워 맞히는 것이 아니라 문제 문구와 공식 법리 컨텍스트로 결론을 도출하게 하려는 목적은 v4와 같습니다.

## 생성 방식

1. `data/bar_exam/round15_rag_contexts_v4_20260625` 전체를 `data/bar_exam/round15_rag_contexts_v5_20260629`로 보존 복사했습니다.
2. 문항 파일 제목의 버전 표기를 `v5`로 바꾸었습니다.
3. 기존 `v4 수동 보강 포인트` 섹션명은 `v5 수동 검증 보강 포인트`로 바꾸었습니다.
4. 법리 본문은 우선 그대로 유지했습니다. 확인 없이 맞던 문항을 다시 쓰지 않는 것이 원칙입니다.
5. 이후 검증은 전 문항을 대상으로 하되, 수정은 리트리버 충돌ㆍ근거 누락ㆍ선지 매칭 위험이 확인된 곳에만 최소 적용합니다.

## v5 운영 원칙

- 전 문항을 검증 대상에 포함합니다.
- 이미 안정적으로 맞은 문항은 원칙적으로 수정하지 않습니다.
- 수동 보강과 자동 `선지별 근거 후보`가 충돌하면 수동 보강과 공식 URL을 우선합니다.
- 자동 후보가 너무 길거나 엉뚱한 판례를 끌어와 모델을 흔드는 경우, 해당 문항만 정리합니다.
- `옳은 것`/`옳지 않은 것`, O/X 집합, 선택지 번호 매칭은 법리 근거와 별도로 검증합니다.
- 공식 선택 번호는 컨텍스트 파일에 넣지 않습니다.

## 문항별 컨텍스트 품질 기준

각 `qNNN_과목.md`는 그 파일만 보고도 모델이 지문별 O/X를 판단할 수 있어야 합니다.

- 공식 선택 번호, 평가표, 기존 풀이 결과는 넣지 않습니다.
- 문제 방향과 선택지 원문은 보존합니다.
- 법률뿐 아니라 필요한 시행령, 시행규칙, 대법원규칙, 대통령령, 총리령, 부령 등 하위규범을 포함합니다.
- 판례는 문항의 지문 판단에 필요한 요지와 핵심 본문을 우선합니다.
- 자동 검색 후보가 문제와 무관하거나 수동 보강과 충돌하면 그대로 방치하지 않습니다.
- 원칙과 예외가 같이 문제 되는 경우, 예외 요건과 적용 시점을 명시합니다.
- 조합형 문항은 법리 결론과 선택지 번호 매칭을 별개 절차로 검산할 수 있게 둡니다.

## 컨텍스트 제작ㆍ검증 근거 우선순위

이 우선순위는 v5 컨텍스트를 만들고 검증하는 사람에게 적용됩니다. 실제 풀이 실행에서는 로컬 파일이나 웹을 열 수 없으므로, 필요한 근거는 문항별 RAG 파일 안에 들어 있어야 합니다.

1. 해당 문항의 `v5 수동 검증 보강 포인트`
2. 문항 파일 안의 법령ㆍ명령ㆍ규칙 원문과 판례 원문
3. 로컬 법령 md: `/home/work/.projects/LLM-OS-Models/Terminal/HRM-Text/legalize-kr`
   - 법률, 시행령, 시행규칙, 대법원규칙, 총리령, 부령, 대통령령 등 문제 해결에 필요한 하위규범을 함께 확인합니다.
4. 로컬 판례 md: `/home/work/.projects/LLM-OS-Models/Terminal/precedent-kr`
5. RAG 파일의 공식 URL: 국가법령정보센터, 대법원, 헌법재판소
6. 추가 웹 검색이 필요한 경우에도 공식 법령ㆍ판례ㆍ헌법재판소 자료만 사용

블로그, 학원 해설, 공식 번호표는 컨텍스트 보강 근거로 사용하지 않습니다.

## 사용 우선순위

문항을 풀릴 때는 해당 `qNNN_과목.md` 전체를 넣되, 아래 순서를 지키게 해야 합니다.

1. 문제 방향을 먼저 확정합니다: `옳은 것`, `옳지 않은 것`, `O/X 조합`, `개수형`.
2. `v5 수동 검증 보강 포인트`가 있으면 자동 판례 후보보다 우선 적용합니다.
3. 각 지문을 `ㄱ/ㄴ/ㄷ/ㄹ/ㅁ` 단위로 독립 판단하고, O/X 표를 먼저 만듭니다.
4. 자동으로 붙은 `판례 근거`와 `선지별 근거 후보`는 보조자료로만 사용합니다.
5. 자동 후보와 수동 보강이 충돌하면 수동 보강과 공식 URL을 우선합니다.
6. 마지막에 산출된 O/X 집합을 선택지 번호와 하나씩 대조합니다.
7. 선택한 번호의 지문 집합이 산출된 집합과 정확히 일치하는지 최종 검산합니다.
