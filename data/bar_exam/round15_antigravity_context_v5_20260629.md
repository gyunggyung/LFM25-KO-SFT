# Antigravity 제15회 변호사시험 풀이 지시문 v5

이 지시문은 Antigravity에 그대로 넣기 위한 제15회 변호사시험 풀이 컨텍스트다. v5는 v4 기반 안정화판이며, 핵심 목표는 문제 원문과 v5 RAG 컨텍스트만으로 150문항을 직접 풀게 하는 것이다.

## 사용할 자료

- 문제 원문 JSON
  - `15th_split/public_law`
  - `15th_split/criminal_law`
  - `15th_split/civil_law`
- RAG 컨텍스트
  - `round15_rag_contexts_v5_20260629`
- 출력 폴더
  - `15th_solved_v5/public_law`
  - `15th_solved_v5/criminal_law`
  - `15th_solved_v5/civil_law`

`round15_rag_contexts_v5_20260629`의 `## 문제`와 `## 선지 분해`는 `15th_split` JSON 원문을 기준으로 정리되어 있다. 그래도 풀이할 때는 항상 `15th_split`의 문제 원문을 먼저 읽고, 같은 번호의 v5 컨텍스트로 근거를 보강한다.

## 금지 자료

풀이 실행 중에는 아래 자료를 열람하지 않는다.

- 제15회 공식 정답, 채점표, 해설, 학원 해설, 블로그 해설
- 기존 제15회 solved 파일 또는 이미 풀린 답안
- `round15_rag_verified`처럼 정답이나 검증 결과가 들어 있을 수 있는 폴더
- 로컬 `legalize-kr`, `precedent-kr`, `law-kr`, 기타 원문 법령ㆍ판례 폴더
- 웹 검색, Google 검색, 법령 사이트 직접 접속
- v2, v3, v4 RAG 컨텍스트

v5 컨텍스트 안에 공식 URL이나 로컬 경로가 적혀 있더라도, 그것은 근거 제작 단계의 출처 표시다. 실제 풀이 단계에서는 v5 문서 안에 이미 들어 있는 법령ㆍ명령ㆍ규칙ㆍ판례 내용만 사용한다.

## 근거 우선순위

각 문항의 지문별 O/X는 아래 순서로 판단한다.

1. `15th_split` 문제 원문
2. 같은 번호의 `round15_rag_contexts_v5_20260629/qNNN_과목.md`
3. `## v5 수동 검증 보강 포인트`
4. 같은 q-context 안에 포함된 법령ㆍ명령ㆍ규칙 원문
5. 같은 q-context 안에 포함된 판례 요지와 판례 본문
6. 자동 `선지별 근거 후보`

`## v5 수동 검증 보강 포인트`가 있으면 그 섹션을 최우선으로 적용한다. 자동 후보가 문제와 다른 쟁점, 최신 사후 자료, 하급심 노이즈, 무관한 규칙을 끌어온 경우에는 결론 근거로 쓰지 않는다.

## 필수 풀이 절차

각 문항마다 아래 절차를 생략하지 않는다.

1. `15th_split`에서 문제 JSON을 연다.
2. 같은 과목과 번호의 v5 q-context를 연다.
3. 문제 방향을 먼저 확정한다: `옳은 것`, `옳지 않은 것`, `O/X 조합`, `개수형`.
4. 모든 지문을 `ㄱ`, `ㄴ`, `ㄷ`, `ㄹ`, `ㅁ` 단위 또는 번호 선택지 단위로 분해한다.
5. 각 지문마다 O/X와 근거를 먼저 판단한다.
6. 아래 표를 작성한다.

```markdown
| 지문 | 판단 | 근거 | 확신도 |
|---|---|---|---|
| ㄱ | O/X | 적용 법령ㆍ판례ㆍ법리 | 상/중/하 |
```

7. 문제 방향에 맞는 목표 지문 집합을 적는다.
   - 예: `옳지 않은 것: ㄱ, ㄷ`
   - 예: `옳은 것: ㄴ, ㄷ`
8. 선택지 ①~⑤를 원문 그대로 다시 적고, 각 번호의 지문 집합을 목표 지문 집합과 비교한다.
9. 목표 지문 집합과 정확히 일치하는 번호만 최종 답으로 고른다.
10. 저장 직전에 선택지 번호를 다시 검산한다.

## 저장 형식

각 문항마다 풀이 markdown과 solved JSON을 모두 저장한다. JSON 스키마는 기존 solved 파일의 형식을 참고하되, 제15회 기존 답안 파일은 열람하지 않는다. 스키마가 불명확하면 최소한 아래 필드를 포함한다.

```json
{
  "question_id": "<subject>_<NNN>",
  "answer": "<직접 추론한 1-5 번호>",
  "selected_letters": ["<해당 지문 표지>"],
  "question_direction": "<옳은 것/옳지 않은 것/OX 조합/개수형>",
  "confidence": "<high|medium|low>",
  "reasoning_file": "<subject>_<NNN>_solution.md"
}
```

Markdown 풀이 파일에는 반드시 아래 항목을 둔다.

- 문제 방향
- 지문 판단표
- 목표 지문 집합
- 선택지 대조
- 최종 검산

## 실패 패턴 방지

- `옳은 것`과 `옳지 않은 것`을 뒤집지 않는다.
- ㄱ~ㅁ O/X 판단이 끝나기 전에는 번호를 고르지 않는다.
- 선택지 원문을 기억으로 재구성하지 않는다.
- 같은 법리의 원칙과 예외가 같이 나오면 예외 요건부터 확인한다.
- 시점이 중요한 문항은 행위시, 처분시, 판결확정시, 위헌결정시, 재판계속시를 분리한다.
- 공동피고인, 전문법칙, 공소기각 판결/결정, 포괄일죄, 채권자대위, 변제자대위 문제는 지위와 절차 단계를 분리한다.
- 확신도가 낮아도 반드시 하나를 선택하되, `confidence`를 낮추고 부족한 근거를 풀이 파일에 적는다.

## 실행 지시

제15회 변호사시험 150문항을 과목별로 하나씩 직접 풀고 저장한다. 각 문항마다 O/X 표와 선택지 집합 검산을 반드시 남긴다. 최종 답은 기억이나 기존 답안이 아니라 `15th_split` 문제 원문과 v5 q-context 근거에서 추론한다.
