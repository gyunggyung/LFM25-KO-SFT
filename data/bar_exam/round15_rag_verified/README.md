# 제15회 변호사시험 RAG 참고 문서 (round15_rag_verified)

제15회 변호사시험 150문항 각각에 대해, 문제를 풀기 위한 관련 법령 원문, 판례 발췌, 참고 URL을 정리한 마크다운 파일 모음.

## 폴더 구조

```
round15_rag_verified/
  q001_공법.md       # 제15회 공법 q1
  q002_공법.md
  ...
  q150_형사법.md     # 제15회 형사법 마지막 문항
  README.md          # 본 문서
```

각 파일명: `q{문항번호:03d}_{과목}.md` (예: `q001_공법.md`, `q039_민사법.md`)

## 각 파일의 구성

```markdown
# 제15회 변호사시험 q{N} — [{과목}]

## 문제                 ← 문제 본문 + 5지선다 보기 전체
## 정답                 ← 공식 정답 (단일: 1~5, 조합형: 10/20/30/40/50/60/70)
## 추출된 키워드         ← 문제에서 자동 추출한 법령명, 조문번호
## 관련 법령 (legalize-kr) ← 관련 법령 원문 발췌 + law.go.kr URL
## 관련 판례/행정문서     ← precedent-kr 판례 발췌 (또는 "자동 매칭 없음")
## 추가 참고 URL          ← law.go.kr, 헌법재판소, 대법원 등
```

## 데이터 출처 및 수집 방법

### 1. 문제/정답 데이터

- **출처**: `gyung/korean-bar-exam-moj-multiple-choice` (`data/questions.csv`)
- **다운로드**: `huggingface_hub.hf_hub_download("gyung/korean-bar-exam-moj-multiple-choice", "data/questions.csv", repo_type="dataset")`
- **필터링**: round=15인 150문항 전부 사용 (조합형 정답 10/20/30/40/50/60/70 포함)
- **라이선스**: Korea Open Government License Type 1 (KOGL Type 1), 법무부 제공

### 2. 법령 원문 (legalize-kr)

- **출처**: `legalize-kr/legalize-kr` GitHub 저장소 로컬 스냅샷 (`legalize-kr/kr/`)
- **수집 방법**:
  1. 문제 본문(stem + question_text)에서 법령명 키워드 추출 (정규식 + 사전 매칭)
  2. 법령명 사전: 헌법, 민법, 상법, 형법, 행정소송법 등 50+ 개 한국 법령명
  3. 법령명이 명시되지 않은 경우, 과목별 기본 법령 매핑 사용:
     - 공법 → 대한민국헌법, 헌법재판소법, 행정소송법, 행정심판법 등
     - 민사법 → 민법, 민사소송법, 상법 등
     - 형사법 → 형법, 형사소송법 등
  4. `legalize-kr/kr/{법령명}/법률.md` (또는 `{법령명}.md`)에서 원문 읽기
  5. 문제에서 언급된 조문(제N조)이 있으면 해당 조문 발췌
  6. 항상 법령 본문 처음 30줄도 포함 (전체 구조 파악용)
- **URL**: `https://www.law.go.kr/법령/{법령명}` (국가법령정보센터)

### 3. 판례/행정문서 (precedent-kr / hrm_text_extra)

- **출처 1**: `precedent-kr/` 로컬 디렉토리 (대법원/헌법재판소/하급심 판례 원문)
  - 경로: `precedent-kr/{민사|형사|공법|일반행정|세무}/{대법원|하급심|헌법재판소}/{법원}_{날짜}_{사건번호}.md`
- **출처 2**: `korean_admrule_precedent_raw_full_20260524.jsonl` (1.7GB, 203,515 라인)
  - 행정규칙/훈령/고시/예규 원문
- **출처 3**: `korean_legal_tasks_full_20260524.jsonl` (4.1GB, 1,383,749 라인)
  - 법령 기반 과제 + 판례 요약
- **수집 방법**:
  1. 문제 stem에서 핵심 키워드 3-5개 추출 (예: "사기죄", "죄수", "포괄일죄")
  2. 추출한 키워드를 OR 결합한 정규식 패턴 생성
  3. `grep -E "{pattern}" {jsonl_path}` 로 jsonl 파일에서 매칭 라인 검색
  4. 매칭된 JSON 라인에서 instruction/response 추출
  5. 키워드 매칭 개수(score)로 정렬, 상위 2개씩 선정
  6. path 필드에서 원본 파일 경로 추출 (출처 표시용)

### 4. 조합형 정답 처리

15회 변호사시험의 일부 문항(11개)은 ㄱ/ㄴ/ㄷ/ㄹ 보기를 조합한 다지선다 형식:
- 정답 `30` → 보기 ③ (ㄷ)
- 정답 `40` → 보기 ④ (ㄹ)
- 정답 `70` → 보기 ⑦ (ㄱ,ㄹ 등)

평가 시 `30` → `3`, `40` → `4` 등으로 정규화하여 단일 정답과 동일하게 처리.

## 검증 방법

### 1차 자동 분류

150개 파일을 다음 기준으로 분류:

| 기준 | 정확 (verified) | 검토 필요 (to_fix) |
|------|-----------------|-------------------|
| 판례 매칭 점수 | 점수 ≥ 2 또는 실제 판례(precedent-kr) 매칭 | 점수 = 1 또는 행정문서만 매칭 |
| 법령 매칭 | 관련 법령 원문 있음 | 법령 없음 |
| 행정문서 | 행정문서/훈령이 아닌 실제 판례 포함 | 행정문서(훈령/고시/예규)만 매칭 |

**1차 결과**: 140개 verified, 10개 to_fix

### 2차 수동 검증 (10개 to_fix)

10개 파일에서 행정문서/훈령 매칭 제거, 부정확한 판례 정리:
- 행정문서 path 패턴: `고용노동부/`, `교육부/`, `국가인권위원회/` 등 정부부처 훈령/고시/예규
- 행정문서 매칭 블록(`점수 N | path: ...`)과 해당 발췌 블록(```) 제거
- 실제 판례(precedent-kr/)만 남기거나, 판례 없으면 "자동 매칭 없음 — 수동 검색 필요" 메시지 추가

**2차 결과**: 10개 전부 verified로 이동, 행정문서 제거 완료

### 현재 상태

- **verified**: 150/150 파일
- **to_fix**: 0개
- **법령 매칭**: 150/150 (과목별 폴백 포함)
- **판례 매칭**: 약 130/150 (나머지 20개는 "자동 매칭 없음")
- **행정문서**: 전부 제거됨 (판례와 무관한 훈령/고시)

## 한계 및 추가 작업 필요사항

1. **판례 매칭 정확도**: 키워드 기반이라 문항과 직접 관련 없는 판례가 섞일 수 있음. 정확도 높이려면 LLM 기반 시맨틱 매칭 필요.
2. **약 20개 문항 판례 없음**: jsonl에서 키워드 매칭이 안 된 문항. 수동으로 판례 검색(https://www.law.go.kr/precInfoP.do) 후 추가 가능.
3. **법령 발췌 범위**: 처음 30줄 + 언급 조문. 문제와 직접 관련된 조문 전체가 필요하면 수동으로 발췌 범위 확장 필요.
4. **조합형 정답 해석**: `30`→`3` 매핑이 임의적. 실제 변시 정답지에서 보기-조합 매핑 확인 필요.
